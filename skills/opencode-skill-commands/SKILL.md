# OpenCode Skill Commands

This skill converts installed `SKILL.md` skills into OpenCode custom slash commands.

Use it to generate command wrapper files such as:

```text
~/.config/opencode/commands/code-review.md
```

so the user can type:

```text
/code-review review the current diff
```

instead of manually asking OpenCode to use a skill.

## Workflow

When this skill is used:

1. Discover local skills.
2. Generate one OpenCode command file per skill.
3. Make each command explicitly load and apply its matching skill.
4. Delete stale generated command files when the matching skill no longer exists.
5. Preserve hand-written command files unless the user asks to overwrite.
6. Report created, updated, deleted stale, skipped, and conflicting commands.

## Skill Search Paths

Check these paths, if they exist:

```text
~/.agents/skills/*/SKILL.md
~/.claude/skills/*/SKILL.md
~/.config/opencode/skills/*/SKILL.md
~/.config/opencode/skill/*/SKILL.md
.opencode/skills/*/SKILL.md
.opencode/skill/*/SKILL.md
.claude/skills/*/SKILL.md
.agents/skills/*/SKILL.md
```

By default, generate global commands from all discovered skills. If the user asks for project-local commands, write to `.opencode/commands/`.

## Command Output Paths

Default target:

```text
~/.config/opencode/commands/
```

Project-local target:

```text
.opencode/commands/
```

Ask only if the target is ambiguous. Otherwise default to global commands.

## Generated Command Template

For a skill named `code-review`, generate:

```md
---
description: Use the code-review skill: short description from SKILL.md
---

Apply the `code-review` skill.

$ARGUMENTS
```

If the skill has a useful frontmatter `description`, include a concise version in the command description.

## Sync Rules

- Delete stale generated commands by default when the matching skill no longer exists.
- Never delete hand-written commands.
- Stale deletion only applies to command files that match this skill's generated template.

## Collision Rules

If the command file already exists:

- Skip it by default.
- If it already looks generated by this skill, update it only when the user asks to refresh or overwrite.
- Never delete hand-written commands.
- If a skill name conflicts with a built-in OpenCode command such as `help`, `init`, `undo`, `redo`, `compact`, or `share`, skip it and report the conflict.

## Script

Prefer running the bundled script:

```bash
node ~/.agents/skills/opencode-skill-commands/scripts/generate-opencode-commands.mjs
```

Useful options:

```bash
node ~/.agents/skills/opencode-skill-commands/scripts/generate-opencode-commands.mjs --dry-run
node ~/.agents/skills/opencode-skill-commands/scripts/generate-opencode-commands.mjs --project
node ~/.agents/skills/opencode-skill-commands/scripts/generate-opencode-commands.mjs --overwrite
node ~/.agents/skills/opencode-skill-commands/scripts/generate-opencode-commands.mjs --skills-dir ~/.agents/skills
node ~/.agents/skills/opencode-skill-commands/scripts/generate-opencode-commands.mjs --target-dir ~/.config/opencode/commands
```

## Final Report

Always summarize:

```text
Created: N
Updated: N
Deleted stale: N
Skipped existing: N
Skipped conflicts: N
Target: ~/.config/opencode/commands
```

List command names that were created or updated:

```text
/code-review
/frontend-ui
/git-commit
```
